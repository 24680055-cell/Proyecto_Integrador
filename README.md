REPORTE DE PRÁCTICA (UNIDAD 2)
1. Introducción
En esta práctica se trabajó con el lenguaje de programación Python y la biblioteca Flet para el desarrollo de una interfaz gráfica de usuario (GUI). Flet permite crear aplicaciones interactivas mediante un modelo de controles jerárquicos, facilitando la visualización de datos de manera profesional y estructurada.
2. Objetivo
Desarrollar un catálogo de productos funcional utilizando contenedores dinámicos, organización responsiva mediante filas y el manejo de recursos locales (imágenes), asegurando que el código cumpla con las reglas de sintaxis y estructura de la Unidad 2.

3. Marco Teórico
La aplicación se fundamenta en los siguientes conceptos de Flet:

Controles (Widgets): Elementos visuales como botones, textos e imágenes.
Layout (Row): Sistema de organización horizontal que permite el salto de línea automático con la propiedad wrap.

Contenedores: Elementos que permiten aplicar estilos como rellenos (padding), bordes redondeados y sombras para mejorar la estética.

Estructuras de Datos: Uso de listas y diccionarios en Python para la gestión eficiente de la información del catálogo.

4. Desarrollo
El proceso de construcción del software consistió en:
Modelado de datos: Organización de la información de los productos en diccionarios.

Lógica del Ciclo: Implementación de un bucle for para la generación masiva de tarjetas de productos, optimizando el código.

Configuración de Assets: Definición de la ruta de recursos para enlazar las imágenes locales con la interfaz.

Corrección de Sintaxis: Ajuste de la indentación y bloques de código para asegurar la ejecución correcta en el intérprete de Python.
5. Resultados
Se obtuvo una aplicación de escritorio que despliega un catálogo visualmente atractivo. La interfaz es capaz de adaptarse a diferentes tamaños de ventana y permite la interacción básica a través de botones de compra, demostrando la integración exitosa de la lógica de programación con el entorno gráfico.
